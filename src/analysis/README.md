# Analysis Module

This module provides comprehensive analysis tools for low-voltage (LV) distribution grids.

## Structure

```
src/analysis/
├── core/                    # Core analysis (DB-dependent, for synthetic grids)
│   ├── topology_analysis.py # Topology metrics (ParameterCalculator)
│   └── powerflow_analysis.py # Power flow calculations
│
├── validation/              # Validation suite (standalone, for external/DSO grids)
│   ├── network_adapter.py   # Normalize external networks
│   ├── grid_splitter.py     # Split multi-grid networks
│   ├── metrics_calculator.py # Compute metrics without DB
│   ├── naming_conventions.py # Parse SWD/Forchheim naming
│   ├── config.py            # Configuration management
│   └── README.md            # Detailed validation documentation
│
├── tools/                   # Standalone utilities
│   ├── convert_json_to_excel.py # JSON to Excel converter
│   └── export_geodata_to_csv.py # Export geodata for QGIS
│
├── docs/                    # Documentation and examples
│   ├── grid_validation_examples.ipynb
│   ├── forchheim_naming_convention.md
│   └── config_validation.yaml.template
│
├── utils.py                 # Shared utility functions
└── config_validation.yaml   # Active config (gitignored)
```

## Quick Start

### 1. Analyze External/DSO Network

```python
from src.analysis.validation import NetworkAdapter, MetricsCalculator

# Load your network
import pandapower as pp
net = pp.from_json('/path/to/network.json')

# Adapt to standard structure
adapter = NetworkAdapter(net, naming_convention='auto')
adapted_net = adapter.adapt()

# Compute metrics
calculator = MetricsCalculator()
metrics = calculator.compute_with_estimation(adapted_net)

print(f"Cable length: {metrics['cable_length_km']:.2f} km")
print(f"Households: {metrics['no_households']}")
```

### 2. Split Multi-Grid Network

```python
from src.analysis.validation import GridSplitter

# Load network with multiple grids
net = pp.from_json('/path/to/multi_grid.json')

# Split into individual grids
splitter = GridSplitter(net, method='auto')
grids = splitter.split()

# Save grids
splitter.save_grids(grids, output_dir='/output/path/')
```

### 3. Use Command-Line Interface

```bash
# Configure your network path in config_validation.yaml first
cd /home/breveron/git/github/pylovo

# Split multi-grid network
python runme/run_validation.py split

# Analyze network(s)
python runme/run_validation.py analyze --export

# Export to Excel
python runme/run_validation.py export --format excel
```

## Core vs Validation

### Core Module (Database-Dependent)
- **Purpose**: Analyze synthetic grids generated by PyLovo
- **Database**: Requires PostgreSQL connection
- **Use case**: Grid generation, clustering, parameter analysis
- **Entry point**: `ParameterCalculator` class

### Validation Module (Standalone)
- **Purpose**: Analyze external/DSO grids for benchmarking
- **Database**: No database required
- **Use case**: Validate synthetic grids against real DSO data
- **Entry point**: `MetricsCalculator` class

## Supported Naming Conventions

The validation module automatically detects and parses:

1. **SWD** - Hierarchical naming with underscores (e.g., "7137137_001001_005005")
2. **Forchheim** - Continuous numeric naming (30 digits)
3. **Generic** - Any other network structure

See `docs/forchheim_naming_convention.md` for detailed specifications.

## Computed Metrics

All metrics compatible with CLUSTERING_PARAMETERS:

**Topology**:
- `no_branches` - Number of main feeders
- `no_house_connections` - Consumer buses
- `no_connection_buses` - Internal nodes

**Load**:
- `no_households` - Total households
- `max_power_mw` - Peak power demand
- `simultaneous_peak_load_mw` - Coincident peak load

**Spatial**:
- `cable_length_km` - Total cable length
- `avg_trafo_dis` - Average distance to transformer
- `house_distance_km` - Average distance between houses

**Electrical**:
- `resistance` - Aggregate resistance
- `vsw_per_branch` - Voltage drop per branch
- `transformer_mva` - Transformer rating

## Development

### Adding New Naming Convention

1. Add parser to `validation/naming_conventions.py`:
```python
def parse_custom_chr_name(chr_name: str) -> Dict[str, str]:
    # Your parsing logic
    return {'grid_id': ..., 'objekttyp': ...}
```

2. Update `detect_naming_convention()` to recognize it

3. Add test cases

### Adding New Metrics

1. Add calculation method to `core/topology_analysis.py`
2. Update `CLUSTERING_PARAMETERS` in `config_loader.py`
3. Update `MetricsCalculator._compute_metrics_standalone()`

## Troubleshooting

### "Network has no transformers"
- Check if your JSON contains trafo table
- Verify network loaded correctly

### "calc_resistance failed"
- Network may not be radial
- Non-radial networks get fallback resistance=0

### Import errors
- Ensure you're running from project root
- Check PYTHONPATH includes project root
